# Library-Book-Usage-and-Reading-Pattern-Analysis

Project Title:
"Library Book Usage and Reading Pattern Analysis"

Domain.   : Data Analysis & Visualization
Technology: Python, Flask, MySQL, HTML/CSS
Typeb.    : Individual Mini Project — AIDS, Anna University

Problem Statement :

"Libraries in educational institutions maintain large collections of books, but lack proper tools to track and analyze borrowing patterns. There is no clear understanding of which books are frequently used, which departments are most active, and during what periods library usage is highest. Without this data-driven insight, libraries face issues like poor book inventory management, underutilized resources, and inability to meet student reading demands. This project aims to analyze library book usage data and visualize reading patterns to help institutions make smarter, data-driven decisions for better library management."


Project Objectives :

1.To collect and organize library book borrowing data into a structured database
2.To clean and preprocess the data by removing duplicates and handling missing values
3.To analyze borrowing patterns and identify the most and least frequently borrowed books
4.To examine department-wise and year-wise reading activity among students
5.To identify peak library usage periods such as exam seasons and holidays
6.To calculate the average duration for which students borrow books
7.To visualize the analyzed data using bar graphs, pie charts, line graphs and heatmaps
8.To provide meaningful insights that help librarians make better inventory decisions


Users & Modules :

User's of the system :

1. Admin (Librarian)
The Admin manages the entire library system. Admin can add, update, and delete book records, manage student details, and record borrowing and return transactions.

2. Student
The Student uses the system to borrow and return books and search for book availability in the library.

3. Analyst / Viewer
The Analyst views the analysis reports, charts, and visualizations generated by the system to understand reading patterns and support library decisions.

Modules of the system :

Module 1 — User Management Module
Handles login and authentication for all users — Admin, Student, and Analyst.

Module 2 — Book Management Module
Allows the Admin to add, update, and delete book records in the library database.

Module 3 — Borrowing & Return Module
Records and tracks all book borrowing and return transactions done by students.

Module 4 — Data Cleaning Module
Removes duplicate records, fixes missing values, and prepares data for analysis.

Module 5 — Analysis Module
Analyzes borrowing patterns, peak usage periods, and department-wise reading activity.

Module 6 — Visualization Module
Generates bar charts, pie charts, line graphs, and heatmaps from the analyzed data.

Module 7 — Report Generation Module
Compiles all analysis results into a structured report for the Admin and Analyst.
